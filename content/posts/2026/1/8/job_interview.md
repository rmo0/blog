## Paizaとは
ITエンジニア向け国内最大の転職・就職・学習プラットフォーム  
[公式サイト](https://paiza.jp/)にはこう書かれていました。

最初は就職というよりも、学生時代に言語学習の目的で利用していた覚えがあります。
やー、時の流れって早いですね～  
~~あんまり使ってなかったのは秘密~~

## なぜ就職活動に使うに至ったか
### 就職サービスの現実
正直、僕みたいな人がタウンワークとかそこら辺の転職サービスなどを使ってエンジニアの職種に応募をすると、まず書類選考で落とされます。  
理由は実務経験と学歴がないからです。かといって最近よく見る「未経験からエンジニアを目指そう！！！」みたいなサービスに応募すると、面接させてもらえることはさせてもらいますが、実際に面接で業務内容を聞いてみると、「最初はテレアポから」「カリキュラムをこなす間は別の業務」などそういうのがほとんどです。

**僕はプログラミングがしたいんだ！！！！！！**

いやまじで。
ただでさえテレアポとか立ち仕事が苦手で、プログラミングが大好きでエンジニアの職業に応募しているのに最初は別の業務とか、、モチベなくなるって。  

### Paizaの利点
一方、Paizaは書類選考のかわりに企業側で設定したランクをクリアしないと応募できないというシステムがあります。  
これめっちゃよくないですか？要はスキルチェックとかで勉強しながら自分のランクを上げていってその実力で応募できるんです。
実際、応募した際には企業側にスキルチェックで自分が記述したコードが送信されます。
「AI使ってクリアしてずるできるくね？」って思う人もいると思いますが、Paizaでは24時間不正コードがないかAIで監視されています。
コピペなどしてクリアしたらばれるっていうわけです。
「全部手動で記述すれば突破できるやん。楽勝やなｗ」って人は、試験本番で盛りに盛りまくった嘘の経歴が面接のテストで全てぶっ壊れるのでめっちゃ恥ずかしい思いをすると思います。

### 試験当日  
試験当日はスーツをきた面接官の方が3人ほどいらっしゃいました。
代表の方もいらっしゃったようで結構緊張しました。

最初に軽く自己紹介をしていただき、企業理念や企業情報などの説明をしていただきました。  
僕が面接を受けたところは、電子機器などのマイコンなどを大手から委託してもらって開発している企業でした。募集している言語がC/C++だったところとLinuxの経験があれば尚可と記載されていたことが応募理由です。

自分の経歴を話した後、実際にいくつかご質問をいただき答えていくという質問形式の面接でした。
1時間ほどの面接が終わった後に実際のC言語のテストとSPI診断を30分ずつ受けました。

### テストの手ごたえ
いやーもう本当に悪い癖なんですけど、当日初めての面接で緊張したのかめっちゃ基本の構文をド忘れして脳筋コードになってしまいました。  
具体的な問題内容はここには記載出来ないのですが、あきらかにswitch文を使うことを想定された問題だったのですが、switch文の構文が、JSと混ざってしまい(内容は全く同じ)、行の区切りにコロン(:)をつかうことに対して謎に不安になってしまい安全なif文に逃げてしまいました。あーーだせー

```c++
swicth(event) {
    case 0:
        // (内容)
        break;
    case 1:
        // (内容)
        break;
    default:
        // (内容)
}
```

なんせPythonの構文の書き方と似すぎてて、「こんなコロンを使った書き方C言語であるっけ？？」みたいな不安症が出ました。(よくかんがえればgoto文にある)  
「あーこいつswicth文も知らない初心者か」って思われたくなかったので無理やり判別式にenumをつかってやろうかなと思いました。(結局ダサいのでやめました)  

まあこんな焦り散らかして、SPI診断ではトイレに行きたくてしょうがなかったので計算問題はめちゃくちゃでした。
